## 蒸馏范式（研究 → 任务）
**6 步(把研究变成可落地任务,AI 照此走;不是死流程,是手艺骨架):**
1. **先读怀疑面再读结论** — 先看研究自己的对抗核查/反方,把乐观推荐**打折**(前沿研究系统性夸大)。
2. **抽承重的可证伪主张** — 剥掉 hype,留一句「如果 X 则 Y」+ **适用域**(成立条件/边界) + **证据强度**。
3. **诚实标未验证残余** — 研究**没建立**的明写出来,绝不带进 finding 当真。
4. **落成可落地设计** — 接到**本项目** `file:line` + 复用现有模块 + **设计 §5 对抗测试**(种已知 bug 门必抓,非文档剧场)。落 `findings/`(模板 `findings/_TEMPLATE.md`)。
5. **拆成任务** — 一个 finding 拆成 BOARD 行,每行一个**验收一句话** + 优先级 + 依赖。
6. **溯源回填** — finding↔研究↔任务 写进 `TRACE.md`(防断链 + 覆盖率体检)。

> 硬不变量(对抗测试门必抓 / 诚实标未验证)见 `../RULES.md` §2/§3——蒸馏**继承**,不在此重复。
