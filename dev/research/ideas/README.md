# 研究台 · ideas（创新 / 在研入口 · 探索自由区）

<!-- OS 自带 · 源自 dev-os clone · 勿改。具体 idea 文件用 _TEMPLATE.md 造。 -->

> 论文研读、原创架构设计、RFC、猜想——**还没成熟到能录成任务**的东西先落这里。
> 研究归研究:这里**不挡、不要求严格验收**(写想法不是下注)。

## 放什么
- **论文研读笔记**(一篇 / 一主题一个 `.md`)
- **原创架构设计 / RFC**(提案,未拍板)
- **猜想 / 反思**(对既有设计的质疑、待验证的点子)

## 生命周期
```
ideas/（灵感·RFC·论文笔记）
   ↓ 值得深挖
research/active/<topic>/（在研线程，带工作日志）
   ↓ 蒸馏成熟、build-ready
research/findings/（可落地设计 + 对抗测试要点）
   ↓ 录成任务
tasks/BOARD.md（T-xxx）→ 治理漏斗（验收/门）
```

## 纪律
- 这里是**探索自由区**。只有要「下注」(影响产品/治理)时才进治理漏斗。
- 想法升级为在研线程 → 移/拷到 `../active/<topic>/`。
- 模板见 `_TEMPLATE.md`。
